# 安全指南

## 重要提醒 ⚠️

**切勿将 API 密钥或敏感信息提交到 Git 仓库！**

## 环境变量配置

本项目使用环境变量来管理敏感信息。请按照以下步骤配置：

### 1. 创建 .env 文件

```bash
cp .env.example .env
```

### 2. 填入你的实际密钥

编辑 `.env` 文件：
```
OPENAI_API_KEY=sk-proj-your-actual-key-here
```

### 3. 确认 .env 已被忽略

`.env` 文件已经在 `.gitignore` 中，确保它不会被提交到版本控制系统。

## 如果 API 密钥已泄露

如果你不小心将 API 密钥提交到了 Git 仓库：

1. **立即轮换密钥**：在 OpenAI 控制台中撤销旧密钥并生成新密钥
2. **清理 Git 历史**：使用 `git filter-branch` 或 BFG Repo-Cleaner 清理历史记录
3. **检查是否有未授权使用**：查看 API 使用日志

## 最佳实践

- ✅ 使用环境变量存储所有敏感信息
- ✅ 定期轮换 API 密钥
- ✅ 为不同环境使用不同的密钥（开发/测试/生产）
- ✅ 限制 API 密钥的权限和使用范围
- ❌ 永远不要在代码中硬编码密钥
- ❌ 不要在日志中打印敏感信息
- ❌ 不要将 .env 文件提交到版本控制

## 报告安全问题

如果发现安全漏洞，请通过私密渠道报告，不要公开披露。
